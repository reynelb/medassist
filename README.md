# MedAssist

AI-powered telemedicine platform. Doctors host video consultations with patients, audio is recorded and transcribed via Whisper, and a structured medical report is auto-generated by an LLM for the doctor to review and confirm.

Built as a final-year dissertation project.

---

## Tech stack

| Layer | Technology |
|---|---|
| Frontend | React 19 + Vite + React Router v6 |
| State management | Zustand |
| HTTP client | Axios (with JWT interceptor + auto-refresh) |
| Backend | FastAPI (Python 3.11+) |
| ORM | SQLAlchemy 2.0 (async) |
| Migrations | Alembic |
| Database | PostgreSQL 15 |
| Auth | JWT (python-jose) + bcrypt (passlib) |
| Transcription | OpenAI Whisper API |
| Report generation | OpenAI GPT-4o |
| Video calls | Daily.co (WebRTC) |
| File storage | Local filesystem (dev) / S3 (prod) |

## Setup

### Prerequisites
- Python 3.11+
- Node.js 20+
- PostgreSQL 15+

### 1. Clone and enter the repo
```bash
git clone https://github.com/YOUR_USERNAME/medassist.git
cd medassist
```

### 2. Database
```bash
createdb medassist
```

### 3. Backend
```bash
cd backend
cp .env.example .env
# Edit .env with your DB credentials and API keys

pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload --port 8000
```

API docs: http://localhost:8000/docs

### 4. Frontend
```bash
cd frontend
cp .env.example .env
npm install
npm run dev
```

App: http://localhost:5173

## User roles

| Role | Access |
|---|---|
| **Doctor** | Dashboard, all patients, appointments, call room (with recording), transcripts, reports, invitations, own profile |
| **Patient** | Home, assigned doctor profile, own appointment history, call room (join only), own profile |

Patients **cannot** see transcripts or medical reports. Role is enforced on both frontend (route guards) and backend (FastAPI dependency injection).

---

## Core data flow

```
1. Doctor creates appointment (POST /appointments/)
2. Doctor starts call (POST /appointments/:id/start)  →  room_id returned
3. Both join video call (Daily.co room)
4. Doctor ends call (POST /appointments/:id/end)
5. Frontend uploads audio blob (POST /transcripts/calls/:id/recording)
6. Backend runs Whisper in background task  →  transcript saved
7. Backend sends transcript to GPT-4o  →  report draft saved
8. Doctor polls GET /reports/:id until status = draft
9. Doctor edits report fields  (PATCH /reports/:id)
10. Doctor confirms report (POST /reports/:id/confirm)  →  status = confirmed
```

## Dissertation context

This project is submitted as a final-year Computer Science dissertation. The platform demonstrates the integration of real-time communication (WebRTC), automatic speech recognition (Whisper), and large language models (GPT-4o) within a clinically aware two-role web application.
